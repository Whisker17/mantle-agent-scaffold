## DeFi Read Source Policy

Intent routing (v0.2.7):

- `pool_quote` (`mantle_getSwapQuote`): DexScreener provider route first; in `best` mode both providers are queried and ranked.
- `pool_liquidity` (`mantle_getPoolLiquidity`): DexScreener pair first, then subgraph fallback, then indexer SQL fallback.
- `pool_opportunity_scan` (`mantle_getPoolOpportunities`): DexScreener token-pairs endpoint for candidate pool discovery and ranking.
- `protocol_tvl` (`mantle_getProtocolTvl`): DefiLlama protocol TVL first, then subgraph fallback, then indexer SQL fallback.
- `protocol_lending_markets` (`mantle_getLendingMarkets`): on-chain Aave first, then subgraph/indexer fallbacks.

Fallback trigger policy:

- A lower tier is used only when upper tier is empty, errored, or missing required fields.
- Successful higher-tier data is preserved.

Valuation policy:

- Pool USD valuation path is DexScreener token prices first, DefiLlama as fallback.
- Null semantics: unavailable valuation stays `null` instead of `0`.

Conflict and confidence policy:

- Liquidity USD conflict threshold is 20%; above threshold, tool emits `total_liquidity_usd_range`.
- DeFi read outputs emit `source_trace[]` and a normalized `confidence` score bundle.

## Safety and Contract Notes

- Indexer tools enforce endpoint policy (`https`, allowlist/domain checks, private-host blocking).
- SQL interface blocks mutation statements (`insert`, `update`, `delete`, `drop`, etc.).
- `mantle_getTokenPrices` never fabricates values; unknown valuations remain `null`.
- Diagnostics probe method is restricted to:
  - `eth_chainId`
  - `eth_blockNumber`
  - `eth_getBalance`

## E2E Coverage Notes

- All 19 tools are represented in scenario registration.
- Indexer scenarios are optional and activated by endpoint env vars (`E2E_SUBGRAPH_ENDPOINT`, `E2E_SQL_ENDPOINT`).
- `mETH` resolution scenario uses `require_token_list_match=false` to reduce external token-list availability flakiness in CI-style runs.
