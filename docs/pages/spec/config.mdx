# Configuration and Operations

## Root Project (MCP)

Common root scripts:

- `npm run build`
- `npm test -- --run`
- `npm run typecheck`
- `npm run test:e2e`

## MCP Runtime Environment Variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `MANTLE_MCP_TRANSPORT` | `stdio` | Transport selector (`stdio` only in current implementation) |
| `MANTLE_RPC_URL` | `https://rpc.mantle.xyz` | Mainnet RPC override |
| `MANTLE_SEPOLIA_RPC_URL` | `https://rpc.sepolia.mantle.xyz` | Sepolia RPC override |
| `MANTLE_TOKEN_LIST_URL` | `https://token-list.mantle.xyz` | Canonical token list source for token resolution cross-check |
| `MANTLE_TOKEN_LIST_PIN_HASH` | unset | Optional token-list integrity pin |
| `MANTLE_TOKEN_LIST_TTL_SECONDS` | `300` | Token-list cache TTL |
| `MANTLE_ALLOWED_ENDPOINT_DOMAINS` | unset | Optional endpoint host allowlist for indexer/diagnostic tools |
| `MANTLE_ALLOW_HTTP_LOCAL_ENDPOINTS` | `false` | Allow `http://localhost` only for local debugging |
| `MANTLE_INDEXER_MAX_ROWS` | `1000` | Max SQL rows returned by `mantle_queryIndexerSql` |
| `MANTLE_SUBGRAPH_MAX_RESPONSE_BYTES` | `1048576` | Max accepted subgraph response payload size |

## E2E Environment Variables

Required:

- `E2E_LLM_PROVIDER`
- `E2E_LLM_API_KEY`

Optional:

- `E2E_LLM_MODEL`
- `E2E_TIMEOUT_MS`
- `E2E_MAX_RETRIES`
- `E2E_OPENROUTER_SITE_URL`
- `E2E_OPENROUTER_APP_NAME`
- `E2E_SUBGRAPH_ENDPOINT`
- `E2E_SQL_ENDPOINT`

Meaning:

| Variable | Purpose |
| --- | --- |
| `E2E_LLM_PROVIDER` | Select provider: `openai`, `anthropic`, or `openrouter` |
| `E2E_LLM_API_KEY` | API key used by selected provider |
| `E2E_LLM_MODEL` | Override default model name for selected provider |
| `E2E_TIMEOUT_MS` | Per-scenario generation timeout |
| `E2E_MAX_RETRIES` | Retry count for retryable scenario failures |
| `E2E_OPENROUTER_SITE_URL` | Optional `HTTP-Referer` sent to OpenRouter |
| `E2E_OPENROUTER_APP_NAME` | Optional `X-Title` sent to OpenRouter |
| `E2E_SUBGRAPH_ENDPOINT` | GraphQL endpoint for optional subgraph scenario |
| `E2E_SQL_ENDPOINT` | SQL endpoint for optional SQL scenario |

## Important Operational Note

`npm run test:e2e` does **not** auto-load `.env`.
Use shell export/source before running.

Typical command sequence:

```bash
set -a
source .env
set +a
unset E2E_SUBGRAPH_ENDPOINT E2E_SQL_ENDPOINT
npm run test:e2e
```
